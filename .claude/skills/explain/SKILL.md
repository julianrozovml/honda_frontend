## 1. Project Overview
Honda Motos Colombia official website. Built with **Next.js 16** (App Router), **React 19**, **TypeScript 6**. Package manager: **pnpm**. Styling: **SCSS only** (Tailwind CSS v4 is installed but not used — all styles are written in SCSS modules or global SCSS).

## 2. Stack & Key Libraries
- **Next.js 16 App Router** — RSC by default, `"use client"` only when needed
- **React 19**
- **TypeScript 6**
- **SCSS** — CSS Modules for `ui/` and `drupal-components/`, BEM global SCSS for `layouts/`
- **shadcn/ui** — base primitive components live in `components/ui/*.tsx` (flat, generated by CLI). Project components wrap these primitives.
- **next/font/google** — Figtree (primary) and Manrope, injected as CSS variables `$font-figtree` / `$font-manrope`
- **Import alias** `@/*` resolves to project root

## 4. Design System (variables.scss)
- **Primary:** `$color-primary-red` (#cc0000), `$color-primary-black` (#1d1d1d), `$color-primary-white` (#ffffff)
- **Dark scale:** `$color-dark-100` → `$color-dark-1000` + transparencies (`$color-dark-10/30/50`)
- **Light scale:** `$color-light-100` → `$color-light-1000` + transparencies
- **Secondary (reds):** `$color-secondary-100` → `$color-secondary-1000` + transparencies
- **Max width:** `$max-width: 1280px`
- **Breakpoints (mobile-first):** xs 320px · s 375px · m 425px · sm 640px · md 768px · lg 1024px · xl 1280px
- Always use `@include md`, `@include lg`, etc. from `mixins.scss`. Never write raw `@media` inside component files.

## 5. Component Architecture Rules
- Every component folder: `ComponentName.tsx` + `ComponentName.module.scss` + `component-name.types.ts`
- File naming: folder **PascalCase**, `.tsx`/`.scss` files **kebab-case** except the module (`ComponentName.module.scss`)
- Props interfaces exported from `.types.ts`, imported with `import type`
- `"use client"` only when the component uses hooks, event handlers, or browser APIs
- Server Components can `await` directly — used for `getIsMobile()` to conditionally exclude images from HTML on mobile (not just CSS hidden)

## 6. Service / Data Fetching Pattern
- **Server Components** fetch data in the page or layout and pass down as props
- **Client Components** receive data as props or interact via **Server Actions** passed as props (e.g. `onSearch` in `VinSearchSection`)
- Service functions belong in `lib/services/` — never coupled to a component
- `VinSearchSection` is the reference implementation: `onSearch: (vin: string) => Promise<Campaign[]>` is a Server Action defined in `page.tsx` and injected as a prop

## 7. shadcn Integration
- Components generated by `pnpm dlx shadcn@latest add <component>` land in `components/ui/<component>.tsx`
- These are **primitives** — do not modify them directly
- Project components (e.g. `(navigation)/Breadcrumb/breadcrumb.tsx`) import from the primitives and apply project styles via `className` props
- `lib/utils.ts` exports the `cn()` helper used by shadcn components
